# Java AI RAG Agent Platform

[中文说明](README_CN.md)

A Java AI application engineering portfolio project built incrementally with Java and Spring Boot.

## Current Iteration

Iteration 2 connects the existing chat endpoint to the DeepSeek API:

- Java 17 and Maven
- Spring Boot 3.5
- Spring AI 1.1
- DeepSeek OpenAI-compatible chat API
- Configurable system prompt for company instructions
- Health check, request validation, and consistent API errors
- Automated tests that never call the real model API

Streaming output, document processing, RAG, Tool Calling, and MCP are not implemented yet.

## Requirements

- Java 17 or later
- Maven 3.9 or later
- A valid DeepSeek API key

## DeepSeek Configuration

The application reads model settings from environment variables. Never commit a real API key.

| Environment variable | Required | Default | Description |
| --- | --- | --- | --- |
| `AI_PROVIDER` | No | `deepseek` | Logical model provider name |
| `DEEPSEEK_API_KEY` | Yes | none | DeepSeek API credential |
| `DEEPSEEK_BASE_URL` | No | `https://api.deepseek.com` | DeepSeek API base URL |
| `DEEPSEEK_MODEL` | No | `deepseek-v4-flash` | Chat model name |
| `AI_SYSTEM_PROMPT` | No | general assistant prompt | Instructions applied to every chat |
| `SERVER_PORT` | No | `8080` | Local HTTP port |

PowerShell example:

```powershell
$env:DEEPSEEK_API_KEY = "your-new-api-key"
$env:AI_SYSTEM_PROMPT = "You are a professional company assistant."
mvn spring-boot:run
```

For IntelliJ IDEA, open **Run/Debug Configurations**, select `AiRagAgentPlatformApplication`, and add `DEEPSEEK_API_KEY` under **Environment variables**.

The `.env.example` file is only a reference. Spring Boot does not load `.env` automatically.

## Run Locally

Run tests without making any real model request:

```bash
mvn clean test
```

Start the application after configuring the API key:

```bash
mvn spring-boot:run
```

## API Endpoints

### Health Check

```http
GET /api/health
```

### Chat

```http
POST /api/chat
Content-Type: application/json

{
  "message": "Explain dependency injection in Java."
}
```

Response:

```json
{
  "answer": "The answer generated by DeepSeek"
}
```

Blank messages return HTTP `400`. Model provider failures return HTTP `502` with the error code `MODEL_PROVIDER_ERROR`.

## Project Structure

```text
src/main/java/com/example/airagagentplatform
|-- config
|   |-- AiChatConfiguration.java  # Creates the Spring AI ChatClient
|   `-- AiChatProperties.java     # Application-level AI settings
|-- controller                    # REST endpoints and API errors
|-- domain                        # Core domain models added later
|-- dto                           # API request and response records
|-- repository                    # Persistence components added later
`-- service
    |-- ChatService.java          # Provider-independent chat contract
    `-- DeepSeekChatService.java  # DeepSeek implementation
```
